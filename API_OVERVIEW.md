# Огляд API: Онлайн-планувальник подорожей

Цей документ описує вигаданий приклад REST API для онлайн-планувальника подорожей.  
API дозволяє керувати подорожами користувача: переглядати список поїздок, дивитися деталі окремої подорожі та створювати нові подорожі.

---

## Огляд ендпоінтів

Умовно передбачено такі ендпоінти:

- `GET /api/trips` — отримати список подорожей.
- `GET /api/trips/{id}` — створювати нову подорож.
- `POST /api/trips` — переглядати деталі однієї подорожі.

Нижче наведено приклади запитів та відповідей для кожного ендпоінта.

---

## 1. Отримати список подорожей

**Метод:** `GET`  
**Endpoint:** `/api/trips`  

Повертає список усіх подорожей поточного авторизованого користувача.

### Приклад запиту

```http
GET /api/trips HTTP/1.1
Host: api.travel-planner.example.com
Authorization: Bearer <token>
```

### Приклад відповіді
```json
[
  {
    "id": "1",
    "destination": "Paris",
    "startDate": "2025-12-01",
    "endDate": "2025-12-10"
  },
  {
    "id": "2",
    "destination": "Copenhagen",
    "startDate": "2026-01-15",
    "endDate": "2026-01-20"
  }
]
```
## 2. Створити нову подорож.

**Метод:** `POST`  
**Endpoint:** `/api/trips`

### Приклад запиту

```http
POST /api/trips HTTP/1.1
Host: api.travel-planner.example.com
Content-Type: application/json
Authorization: Bearer <token>
```

### Приклад тіла запиту

```json
{
  "title": "Італія – літня відпустка",
  "country": "Італія",
  "startDate": "2025-07-01",
  "endDate": "2025-07-10",
  "budgetPlanned": 1200,
  "description": "Подорож по Риму, Неаполю та Амальфі."
}
```

### Приклад відповіді

```json
{
  "id": "trip_3",
  "title": "Італія – літня відпустка",
  "country": "Італія",
  "startDate": "2025-07-01",
  "endDate": "2025-07-10",
  "budgetPlanned": 1200,
  "description": "Подорож по Риму, Неаполю та Амальфі.",
  "createdAt": "2025-03-01T10:15:00Z"
}

```

## 3. Отримати деталі конкретної подорожі

**Метод:** `GET`  
**Endpoint:** `/api/trips/{id}`  

Повертає список усіх подорожей поточного авторизованого користувача.

### Приклад запиту

```http
GET /api/trips/trip_1 HTTP/1.1
Host: api.travel-planner.example.com
Authorization: Bearer <token>
```

### Приклад відповіді
```json
{
  "id": "trip_1",
  "title": "Прага на вихідні",
  "country": "Чехія",
  "startDate": "2025-04-10",
  "endDate": "2025-04-13",
  "days": [
    {
      "date": "2025-04-10",
      "activities": [
        {
          "time": "10:00",
          "type": "sightseeing",
          "title": "Прогулянка старим містом",
          "estimatedCost": 0

        {
          "time": "19:00",
```

## 4. Коди помилок

Нижче наведено основні HTTP-статуси, які може повертати це API:

| Код | Назва                 | Опис                                                       |
|-----|-----------------------|------------------------------------------------------------|
| 200 | OK                    | Запит виконано успішно, дані повернено в тілі відповіді   |
| 201 | Created               | Нову подорож успішно створено                             |
| 400 | Bad Request           | Некоректні або неповні дані в тілі запиту                 |
| 401 | Unauthorized          | Користувач не авторизований або токен недійсний           |
| 404 | Not Found             | Подорож або запитаний ресурс не знайдено                  |
| 500 | Internal Server Error | Внутрішня помилка сервера, спробуйте виконати запит пізніше|
